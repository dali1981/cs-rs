# Constant-Maturity IV Visualization Scripts

This directory contains Python scripts for visualizing constant-maturity implied volatility data generated by the `cs atm-iv --constant-maturity` command.

## Overview

The constant-maturity IV implementation interpolates implied volatility in variance space to provide IVs at exact target DTEs (7, 14, 21, 30, 60, 90 days), making time series directly comparable across days for better earnings detection and term structure analysis.

## Workflow

### Step 1: Generate Constant-Maturity IV Data

Use the Rust CLI to generate the parquet data:

```bash
# Set your data directory
export FINQ_DATA_DIR=~/polygon/data

# Generate constant-maturity IVs (full year example)
./target/release/cs atm-iv \
  --symbols AAPL \
  --start 2025-01-01 \
  --end 2025-12-31 \
  --output ./cm_output \
  --constant-maturity

# Or use short alias --cm
./target/release/cs atm-iv \
  --symbols AAPL \
  --start 2025-01-01 \
  --end 2025-12-31 \
  --output ./cm_output \
  --cm
```

This creates: `./cm_output/atm_iv_AAPL.parquet`

### Step 2: Visualize the Data

Choose one of the visualization scripts below based on your needs.

## Visualization Scripts

### 1. `plot_cm_basic_stats.py` - Data Inspection & Statistics

Displays basic statistics and coverage metrics without creating plots.

**Usage:**
```bash
uv run python3 plot_cm_basic_stats.py ./cm_output/atm_iv_AAPL.parquet
```

**Output:**
- Column names and data coverage
- Summary statistics for all CM IVs (7d, 14d, 21d, 30d, 60d, 90d)
- Term spread statistics
- Sample data rows

**When to use:** First step to inspect your data and verify it loaded correctly.

## Data Schema

The parquet files contain these constant-maturity fields:

### Rolling TTE Fields (existing)
- `atm_iv_nearest` - IV at nearest expiration (>min_dte)
- `nearest_dte` - DTE of nearest expiration
- `atm_iv_30d` - IV at ~30 DTE (±7 day tolerance)
- `atm_iv_60d` - IV at ~60 DTE (±7 day tolerance)
- `atm_iv_90d` - IV at ~90 DTE (±7 day tolerance)

### Constant-Maturity Fields (new)
- `cm_iv_7d` - IV at exactly 7 DTE (interpolated)
- `cm_iv_14d` - IV at exactly 14 DTE (interpolated)
- `cm_iv_21d` - IV at exactly 21 DTE (interpolated)
- `cm_iv_30d` - IV at exactly 30 DTE (interpolated)
- `cm_iv_60d` - IV at exactly 60 DTE (interpolated)
- `cm_iv_90d` - IV at exactly 90 DTE (interpolated)
- `cm_interpolated` - Whether IVs were interpolated (vs extrapolated)
- `cm_num_expirations` - Number of expirations used for interpolation

### Spread Fields
- `term_spread_30_60` - Rolling 30d - 60d spread
- `term_spread_30_90` - Rolling 30d - 90d spread
- `cm_spread_7_30` - CM 7d - 30d spread (earnings signal!)
- `cm_spread_30_60` - CM 30d - 60d spread
- `cm_spread_30_90` - CM 30d - 90d spread

## CLI Options

The `cs atm-iv` command supports these options:

```bash
cs atm-iv --help

Options:
  --symbols <SYMBOLS>           Symbol(s) to analyze (comma-separated)
  --start <START>               Start date (YYYY-MM-DD)
  --end <END>                   End date (YYYY-MM-DD)
  --maturities <MATURITIES>     Target maturities in days (default: 7,14,21,30,60,90)
  --tolerance <TOLERANCE>       Maturity tolerance in days (default: 7)
  --output <OUTPUT>             Output directory for parquet files
  --minute-aligned              Use minute-aligned IV computation
  --constant-maturity, --cm     Use constant-maturity interpolation ⭐
  --min-dte <MIN_DTE>           Minimum DTE for expiration inclusion (default: 3)
```

## Key Features

### 1. Constant-Maturity Interpolation
- **True DTEs**: IVs at exact 7/14/21/30/60/90 DTE (not approximate)
- **Variance space**: Interpolates σ²T for mathematical correctness
- **Complete coverage**: Fills gaps from sparse expirations

### 2. Earnings Detection
- **Front-week spread**: `cm_spread_7_30` shows 7d-30d IV elevation
- **Strong signals**: Spread >10pp indicates imminent earnings
- **Moderate signals**: Spread 5-10pp suggests upcoming event

### 3. Term Structure Analysis
- **Backwardation**: Short-term IV > long-term IV (cm_spread_30_60 > 0)
- **Contango**: Short-term IV < long-term IV (cm_spread_30_60 < 0)
- **Shape changes**: Track term structure evolution

## Tips

1. **Start with inspection**: Always run `plot_cm_basic_stats.py` first to verify data quality
2. **Use simple for quick checks**: `plot_cm_simple.py` is fastest for quick looks
3. **Use comprehensive for research**: `plot_cm_yearly_comprehensive.py` shows everything
4. **Earnings detection**: Look for `cm_spread_7_30 > 10pp` as strong signal
5. **Compare periods**: Generate multiple date ranges and compare term structures

## Troubleshooting

**Q: No CM fields in parquet?**
A: Make sure you used `--constant-maturity` or `--cm` flag when generating data.

**Q: All CM IVs are None?**
A: Check `cm_num_expirations` - if 0, there weren't enough expirations > min_dte.

**Q: Plots look weird?**
A: Verify date range in parquet matches what you expect with `plot_cm_basic_stats.py`.

**Q: Script fails with "column not found"?**
A: Your parquet was generated with rolling mode only. Regenerate with `--cm` flag.
